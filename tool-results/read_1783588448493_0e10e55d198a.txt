     1→     1→# Design System — DOCX Skill
     2→     2→
     3→     3→## Color Philosophy
     4→     4→
     5→     5→GLM uses a **mood-driven dynamic color system** instead of fixed named palettes. Colors are constructed from three dimensions:
     6→     6→
     7→     7→### Three Dimensions of Document Color
     8→     8→
     9→     9→| Dimension | Description | Range |
    10→    10→|-----------|-------------|-------|
    11→    11→| **Temperature** | Warm ↔ Cool | Warm (consulting, education) ↔ Cool (tech, medical) |
    12→    12→| **Weight** | Light ↔ Heavy | Light (resume, proposal) ↔ Heavy (legal, academic) |
    13→    13→| **Energy** | Calm ↔ Active | Calm (official, contract) ↔ Active (report, presentation) |
    14→    14→
    15→    15→### Color Token System
    16→    16→
    17→    17→Every document uses 5 color tokens. These are **computed** based on the document's mood, not selected from a fixed list.
    18→    18→
    19→    19→| Token | Role | Guidance |
    20→    20→|-------|------|----------|
    21→    21→| `primary` | Headings, cover title | Dark, authoritative. Derived from Temperature + Weight. |
    22→    22→| `body` | Body text | Near-black with subtle warmth/coolness. Always high contrast. |
    23→    23→| `secondary` | Captions, footnotes | Mid-tone gray. Legible but visually recessive. |
    24→    24→| `accent` | Table headers, lines, links | The "personality" color. Reflects document Energy. |
    25→    25→| `surface` | Table alternating rows, card backgrounds | Very light tint of accent or neutral. |
    26→    26→

   157→   157→### Visual Profile Color Guidance
   158→   158→
   159→   159→For **Profile B (Visual)** scenes — resume, copywriting, and other non-formal documents — prefer **warm neutral** tones aligned with the "Invisible Precision" design philosophy:
   160→   160→
   161→   161→- **Body text**: Use warm dark neutrals (`#37352F`, `#303030`, `#3A3430`) instead of cool blue-grays. This reduces eye strain.
   162→   162→- **Surface/background**: Use warm near-white (`#F7F7F7`, `#FAFAF8`, `#FBF9F7`) instead of cool tints.
   163→   163→- **Accent colors**: Use sparingly and only for functional differentiation (section headers, key metrics, links). 95% of the document should remain monochromatic (black, white, gray).
   164→   164→- **Tables**: Prefer the **Zebra Stripe** style (see Table Styles §2) — hierarchy through background contrast with minimal borders. Fallback: Horizontal-Only.
   165→   165→
   166→   166→This does NOT apply to Profile A (Formal) scenes (report, academic, contract, official-doc, exam), which must retain pure black `"000000"` body text per regulatory standards.
   167→   167→
   168→   168→### Custom Color Generation
   169→   169→
   170→   170→When the pre-defined recipes don't fit, construct colors using these rules:
   171→   171→
   172→   172→1. **primary**: Start from `hsl(hue, 25-40%, 10-18%)` — dark, desaturated
   173→   173→2. **body**: primary lightened 5-8% — readable dark
   174→   174→3. **secondary**: primary lightened 30-40% — clearly subordinate
   175→   175→4. **accent**: Choose a hue reflecting the domain, `hsl(domainHue, 30-50%, 45-55%)`
   176→   176→5. **surface**: accent desaturated to 5-10%, lightened to 96-98%
   177→   177→
   178→   178→**Contrast check**: body text on white must achieve WCAG AA (≥4.5:1). All recipes above pass this.

   255→   255→## Table Styles
   256→   256→
   257→   257→**Profile routing:** Profile A (Formal: report, academic, contract, exam) → Three-Line Table or Horizontal-Only Table. Profile B (Visual: resume, copywriting) → Zebra Stripe (preferred) or Horizontal-Only Table.
   258→   258→
   259→   259→### 1. Three-Line Table (三线表) — Academic
   260→   260→
   261→   261→Only three horizontal lines: top of table, bottom of header, bottom of table.
   262→   262→
   263→   263→```js
   264→   264→const threeLineTable = new Table({
   265→   265→  width: { size: 100, type: WidthType.PERCENTAGE },
   266→   266→  borders: {
   267→   267→    top: { style: BorderStyle.SINGLE, size: 4, color: "000000" },
   268→   268→    bottom: { style: BorderStyle.SINGLE, size: 4, color: "000000" },
   269→   269→    left: { style: BorderStyle.NONE },
   270→   270→    right: { style: BorderStyle.NONE },
   271→   271→    insideHorizontal: { style: BorderStyle.NONE },
   272→   272→    insideVertical: { style: BorderStyle.NONE },
   273→   273→  },
   274→   274→  rows: [
   275→   275→    new TableRow({
   276→   276→      children: headerCells.map(text => new TableCell({
   277→   277→        children: [new Paragraph({ children: [new TextRun({ text, bold: true, size: 21 })] })],
   278→   278→        borders: {
   279→   279→          bottom: { style: BorderStyle.SINGLE, size: 2, color: "000000" },
   280→   280→          top: { style: BorderStyle.NONE },
   281→   281→          left: { style: BorderStyle.NONE },
   282→   282→          right: { style: BorderStyle.NONE },
   283→   283→        },
   284→   284→        margins: { top: 60, bottom: 60, left: 120, right: 120 },
   285→   285→      })),
   286→   286→    }),
   287→   287→    // ... data rows with all borders NONE
   288→   288→  ],
   289→   289→});
   290→   290→```
   291→   291→
   292→   292→### 2. Zebra Stripe — Data Reports
   293→   293→
   294→   294→```js
   295→   295→function zebraRow(cells, index, palette) {
   296→   296→  return new TableRow({
   297→   297→    children: cells.map(text => new TableCell({
   298→   298→      children: [new Paragraph({ children: [new TextRun({ text, size: 21 })] })],
   299→   299→      shading: index % 2 === 0
   300→   300→        ? { type: ShadingType.CLEAR, fill: palette.surface }
   301→   301→        : { type: ShadingType.CLEAR, fill: "FFFFFF" },
   302→   302→      margins: { top: 60, bottom: 60, left: 120, right: 120 },
   303→   303→    })),
   304→   304→  });
   305→   305→}
   306→   306→```
   307→   307→
   308→   308→### 3. Horizontal-Only — Business (Default)
   309→   309→
   310→   310→```js
   311→   311→const horizontalTable = new Table({
   312→   312→  width: { size: 100, type: WidthType.PERCENTAGE },
   313→   313→  borders: {
   314→   314→    top: { style: BorderStyle.SINGLE, size: 2, color: palette.accent.replace("#","") },
   315→   315→    bottom: { style: BorderStyle.SINGLE, size: 2, color: palette.accent.replace("#","") },
   316→   316→    left: { style: BorderStyle.NONE },
   317→   317→    right: { style: BorderStyle.NONE },
   318→   318→    insideHorizontal: { style: BorderStyle.SINGLE, size: 1, color: "D0D0D0" },
   319→   319→    insideVertical: { style: BorderStyle.NONE },
   320→   320→  },
   321→   321→  rows: [/* header row with accent shading, then data rows */],
   322→   322→});
   323→   323→```
   324→   324→
   325→   325→**⚠️ CRITICAL**: Always set `margins` at the Table or TableCell level. Without margins, text touches cell borders.
   326→   326→
   327→   327→### Table Color Token Derivation
   328→   328→
   329→   329→Each palette in `coverPalettes` provides a `table` object with pre-computed colors for all 3 table styles:
   330→   330→
   331→   331→| Token | Used in | Description |
   332→   332→|-------|---------|-------------|
   333→   333→| `table.headerBg` | Zebra Stripe, Horizontal-Only | Table header background color |
   334→   334→| `table.headerText` | Zebra Stripe, Horizontal-Only | Table header text color (must pass WCAG AA contrast) |
   335→   335→| `table.accentLine` | Three-Line | Top/bottom/header-bottom line color |
   336→   336→| `table.innerLine` | Horizontal-Only | Inner horizontal separator line color |
   337→   337→| `table.surface` | Zebra Stripe | Alternating row background (light tint) |
   338→   338→
   339→   339→**Usage:**
   340→   340→```js
   341→   341→const palette = coverPalettes["DS-1"];
   342→   342→const t = palette.table;
   343→   343→// Three-Line: use t.accentLine for border colors
   344→   344→// Zebra: use t.headerBg, t.headerText, t.surface
   345→   345→// Horizontal: use t.headerBg, t.headerText, t.innerLine
   346→   346→```
   347→   347→
   348→   348→**⚠️ High-saturation accent override**: For DM-1, FG-1, and SN-2, the table colors are intentionally darkened/desaturated relative to the cover accent. Bright accent colors that look good on dark cover backgrounds are too eye-straining on white body pages. Always use `palette.table.*` for tables, never the raw `palette.accent`.

   353→   353→## Cover Page Design System
   354→   354→
   355→   355→### Design Philosophy
   356→   356→
   357→   357→Covers use **7 validated layout recipes** + **parameterized variants** instead of free combination. Each recipe's background, layout, and decoration are visually verified. Differentiation comes from **palette × font size × content variation**.
   358→   358→
   359→   359→**Architecture principle:** All recipes use a **single 16838 outer wrapper table** (one row, exact height). Recipes R1/R2/R3 use **ZERO nested tables** — all decoration is achieved via **paragraph borders** (left/right/top/bottom). This ensures maximum cross-engine stability (MS Office + WPS).
   360→   360→
   361→   361→### Cover Color Palettes (Dark Mode + Light Mode)
   362→   362→
   363→   363→Each palette defines 3 core colors (Background, Primary, Accent) + derived cover/table tokens.
   364→   364→
   365→   365→⚠️ **Disambiguation: `cover.titleColor` is a COLOR value, not title text.** All keys under `cover: { ... }` are **color hex codes** for styling cover text elements. They are NOT text content. The actual title text comes from `config.title`. Never use `P.titleColor` as the `text` parameter of a `TextRun` — it must only be used as the `color` parameter.
   366→   366→
   367→   367→```js
   368→   368→// ✅ Correct — config.title is the text, P.titleColor is the color
   369→   369→new TextRun({ text: config.title, color: P.titleColor })
   370→   370→
   371→   371→// ❌ WRONG — using color value as text content
   372→   372→new TextRun({ text: P.titleColor, color: P.titleColor })  // displays "FFFFFF" as visible text!
   373→   373→```

   501→   501→### ⚠️ Recipe Routing Rules (Replaces Free Selection)
   502→   502→
   503→   503→```js
   504→   504→function selectCoverRecipe(docType, industry, titleLength) {
   505→   505→  // No cover for these types
   506→   506→  if (["contract", "official", "exam", "resume"].includes(docType)) return null;
   507→   507→
   508→   508→  // Academic
   509→   509→  if (docType === "academic") return { recipe: "R5", palette: "ACADEMIC" };
   510→   510→
   511→   511→  // Thesis proposal report (开题报告)
   512→   512→  if (docType === "proposal_report") return { recipe: "R5", palette: "ACADEMIC" };
   513→   513→
   514→   514→  // Lesson plans — R6 editorial for arts/general, R4 for STEM
   515→   515→  if (docType === "lesson_plan" || docType === "lessonplan") {
   516→   516→    const stemKeywords = ["math", "physics", "chemistry", "biology", "science", "tech", "computer", "engineering"];
   517→   517→    if (stemKeywords.some(k => (industry || "").toLowerCase().includes(k))) {
   518→   518→      return { recipe: "R4", palette: "DM-1" };
   519→   519→    }
   520→   520→    // Arts, general, and all other lesson plans → R6 editorial
   521→   521→    return { recipe: "R6", palette: "ED-1" };
   522→   522→  }
   523→   523→
   524→   524→  // Creative/branding/design (formal) → R3 centered card frame
   525→   525→  if (["creative", "branding", "design"].includes(docType)) {
   526→   526→    return { recipe: "R3", palette: "SN-2" };
   527→   527→  }
   528→   528→
   529→   529→  // Cultural/newsletter/internal (casual) → R6 editorial
   530→   530→  if (["cultural", "newsletter", "internal"].includes(docType)) {
   531→   531→    return { recipe: "R6", palette: "ED-1" };
   532→   532→  }
   533→   533→
   534→   534→  // Activity/event planning → R6 editorial
   535→   535→  if (docType === "activity") return { recipe: "R6", palette: "ED-1" };
   536→   536→
   537→   537→  // Trend/research reports in cultural/creative/brand fields → R7 Swiss Tech
   538→   538→  if (docType === "trend_report" || docType === "research_report") {
   539→   539→    if (["cultural", "creative", "brand", "design"].includes(industry)) {
   540→   540→      return { recipe: "R7", palette: "ST-1" };
   541→   541→    }
   542→   542→  }
   543→   543→
   544→   544→  // Formal/business subtypes
   545→   545→  if (docType === "whitepaper") return { recipe: "R2", palette: industry === "finance" ? "IG-1" : "CM-2" };
   546→   546→  if (docType === "consulting") return { recipe: "R2", palette: "MIN-1" };
   547→   547→  if (docType === "proposal" || docType === "plan") return { recipe: "R4", palette: "GO-1" };
   548→   548→
   549→   549→  // Reports — palette by industry, use R1
   550→   550→  if (docType === "report") {
   551→   551→    const paletteMap = {
   552→   552→      finance: "IG-1", consulting: "MIN-1",
   553→   553→      tech: "DM-1", ai: "DM-1",
   554→   554→      education: "WM-1", green: "FG-1",
   555→   555→      medical: "MC-1", government: "GV-1",
   556→   556→    };
   557→   557→    return { recipe: "R1", palette: paletteMap[industry] || "DS-1" };
   558→   558→  }
   559→   559→
   560→   560→  // Default
   561→   561→  return { recipe: "R1", palette: "DS-1" };
   562→   562→}
   563→   563→
   564→   564→// ── Long-title override (applied AFTER initial recipe selection) ──
   565→   565→// Call this after selectCoverRecipe() when the actual title text is known.
   566→   566→function applyLongTitleOverride(result, titleLength) {
   567→   567→  if (!result || !result.recipe) return result;
   568→   568→  // R5 (academic) is never overridden — it has its own calcTitleLayoutMixed()
   569→   569→  if (result.recipe === "R5") return result;
   570→   570→  // R6 (editorial) is designed for short titles only (≤20 chars, single line)
   571→   571→  // Long titles → fall back to R1 (handles long titles best)
   572→   572→  if (titleLength > 20 && result.recipe === "R6") {
   573→   573→    return { recipe: "R1", palette: "WM-1" }; // ED-1 has no dark bg, use WM-1 (warm teal)
   574→   574→  }
   575→   575→  // R3/R4 struggle with long titles → fall back to R1 (same palette)
   576→   576→  if (titleLength > 20 && ["R3", "R4"].includes(result.recipe)) {
   577→   577→    return { recipe: "R1", palette: result.palette };
   578→   578→  }
   579→   579→  // Very long titles: even R2 centered looks scattered → R1 left-aligned
   580→   580→  if (titleLength > 30 && result.recipe === "R2") {
   581→   581→    return { recipe: "R1", palette: result.palette };
   582→   582→  }
   583→   583→  return result;
   584→   584→}
   585→   585→```
